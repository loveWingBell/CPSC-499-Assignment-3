## Make Targets

Run all commands from inside the `file/` directory using your command terminal.

| Command | Description |
|---|---|
| `make all` | Build all four tools, then run all tests |
| `make build` | Compile all four tools without running tests |
| `make test` | Run all tools against every file in `Test Files/` |
| `make clean` | Delete all compiled `.class` files, generated ANTLR sources, and the `test-output/` folder |
| `make build-jp3` | Compile Simple-3.27.0 only |
| `make build-j1` | Compile Simple-J1.0.0 only |
| `make build-j8` | Compile Simple-J8 only (also generates ANTLR sources from `.g4` files) |
| `make build-j12` | Compile Simple-J1.2 only (also generates ANTLR sources from `.g4` files) |
| `make test-jp3` | Test Simple-3.27.0 against all test files |
| `make test-j1` | Test Simple-J1.0.0 against all test files |
| `make test-j8` | Test Simple-J8 against all test files |
| `make test-j12` | Test Simple-J1.2 against all test files |

---

## Usage

**Build and test everything at once:**
```bash
make all
```

**Build first, then test separately:**
```bash
make build
make test
```

**Build and test a single tool:**
```bash
make build-j8
make test-j8
```

**Clean up all generated files:**
```bash
make clean
```

## Test Output

After running `make test`, results are saved to `test-output/`, organized by tool. Each test file produces one `.txt` output file per tool. For example, running `Test1_Valid.java` through all four tools produces:

```
test-output/jp3_27/Test1_Valid.java.txt
test-output/j1/Test1_Valid.java.txt
test-output/j8/Test1_Valid.java.txt
test-output/j12/Test1_Valid.java.txt
```

Each output file contains the number of invocations found followed by a line-by-line listing:

```
3 method/constructor invocation(s) found in the input file(s)

System.out.println("Hello"): file Test1_Valid.java, line 5, column 9
new Scanner(System.in): file Test1_Valid.java, line 6, column 16
scanner.nextLine(): file Test1_Valid.java, line 7, column 22
```

If a file contains syntax errors the tool cannot parse, an error message is written to the output file instead.

## Customizing the ANTLR JAR Path

If your ANTLR JAR has a different name or location, override it on the command line:

```bash
make ANTLR_JAR=/path/to/antlr-4.13.1-complete.jar
```

Or update the `ANTLR_JAR` variable directly at the top of the Makefile:

```makefile
ANTLR_JAR ?= $(CURDIR)/antlr-4.13.1-complete.jar
```
